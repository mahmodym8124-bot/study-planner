# Test Execution Scripts - Complete Index

This directory contains comprehensive test execution scripts that will:

1. Execute 5 npm/npx commands in sequence
2. Capture exit codes and full output
3. Probe health endpoints during test:e2e
4. Log all results to timestamped files

## Quick Start

### Easiest Option (Recommended)
```
Double-click: RUN_TESTS.bat
```
This opens an interactive menu to choose your preferred execution method.

### Direct Execution (Batch)
```batch
cd C:\Users\RA store\study-planner
EXECUTE_TESTS_FULL.bat
```

### Direct Execution (PowerShell)
```powershell
cd "C:\Users\RA store\study-planner"
powershell -ExecutionPolicy Bypass -File execute_tests.ps1
```

## Commands Executed

All scripts execute these 5 commands in order:

```
1. npm.cmd install --save-dev @playwright/test
   → Installs Playwright test framework

2. npm.cmd run lint
   → Runs ESLint code quality checks

3. npm.cmd run build
   → Builds Vite frontend for production

4. npx playwright test --list
   → Lists all available Playwright tests

5. npm.cmd run test:e2e
   → Runs end-to-end tests with health probing
```

After execution, check these files for results:

### TEST_EXECUTION_LOG.txt
Main execution log containing:
- Exact commands executed
- Execution timestamps

Health endpoint probes captured during test:e2e:
- Probe timestamps
- HTTP status codes
- Response body snippets
- Error messages (if any)
- Captured for both endpoints:
  - http://localhost:8091/api/health (API Server)
  - http://localhost:5173/api/health (Web Server)

## Troubleshooting

### Port Already in Use
```batch
netstat -ano | findstr :8091
taskkill /PID <PID> /F

netstat -ano | findstr :5173
taskkill /PID <PID> /F
```

### MongoDB Connection Error
Ensure MONGODB_URI is set in .env:
```
MONGODB_URI=mongodb+srv://...
```

### Node/NPM Not Found
Verify installation:
```batch
node --version
npm --version
```

### Script Won't Run
Try running with explicit interpreter:
```batch
REM For batch:
cmd.exe /c EXECUTE_TESTS_FULL.bat

REM For PowerShell:
powershell -ExecutionPolicy Bypass -File execute_tests.ps1

REM For Python:
python.exe execute_tests.py

REM For Node:
node.exe execute_tests.js
```

## Customization

To modify any script:
1. Open with a text editor (Notepad, VS Code, etc.)
2. Find the configuration section
3. Adjust variables like:
   - `REPO_DIR` - Working directory
   - `maxProbes` / `max_probes` - Number of health probes
   - `probeIntervalSeconds` / `probe_interval` - Probe frequency
   - Timeout limits
4. Save and re-run
