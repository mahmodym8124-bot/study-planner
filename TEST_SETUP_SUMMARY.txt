## Output Captured

All scripts generate two log files:

### TEST_EXECUTION_LOG.txt
Contains:
- Exact command strings executed
- Execution timestamps
- **Complete stdout** for each command
- **Complete stderr** for each command
- **Exit code** for each command
- Raw error output for any failures
- Command execution time

### HEALTH_PROBES.txt
Contains:
- Probe timestamps (YYYY-MM-DD HH:MM:SS.fff format)
- HTTP status codes received
- Response body snippets (first 150 characters)
- Error messages if endpoints unreachable
- Port numbers (8091 for API, 5173 for Web)

---

## Probe Details

**During test:e2e execution**, both scripts will:

1. Start concurrent health probing after 3-second delay (allowing servers to boot)
2. Probe both endpoints every 5 seconds
3. Run maximum 15 probes (~75 seconds total)
4. Record:
   - Timestamp
   - HTTP status code
   - Response body (first 100-150 chars)
   - Any errors/timeouts
5. Stop automatically when test:e2e completes

**Endpoints probed**:
```
http://localhost:8091/api/health  (API Server)
http://localhost:5173/api/health  (Web Server)
```

## FAQ

**Q: Will all 5 commands run even if one fails?**
A: Yes, all scripts continue execution regardless of exit codes.

**Q: How long will the whole process take?**
A: Typically 5-15 minutes depending on build and test speed.

**Q: What if ports 8091 or 5173 are already in use?**
A: Kill the existing process using:
```batch
netstat -ano | findstr :8091
taskkill /PID <PID> /F
```

**Q: Can I modify the probe frequency?**
A: Yes, edit the script and change `probeIntervalSeconds` or `probe_interval` variable.

**Q: Where are the log files?**
A: Both in `C:\Users\RA store\study-planner/`:
- TEST_EXECUTION_LOG.txt
- HEALTH_PROBES.txt

**Q: What if MongoDB isn't running?**
A: The test:e2e might fail. Ensure MongoDB is accessible via MONGODB_URI env var.
